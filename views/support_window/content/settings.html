3.2.5 Configuraciones — Panel de programas y dependencias

¿Qué es este panel?

El panel Configuraciones muestra el estado de las herramientas instaladas y las dependencias del software. Al abrirlo la aplicación realiza una comprobación automática del entorno y presenta para cada programa su nombre, descripción breve, ruta en disco y un estado que indica si está Instalado o si aparece un control para Instalar. En la parte superior hay un acceso rápido para abrir el Manual de usuario.

Vista general del panel

Elementos clave visibles en pantalla:

  • Botón Manual de Usuario (arriba): abre la ventana del manual de usuario (Support Window).

  • Sección “Programas instalados”: lista de las herramientas integradas (FastQC, Trimmomatic, SortMeRNA, Kraken2, etc.). Para cada item se muestra:

    • icono y nombre de la herramienta,

    • breve descripción,

    • la ruta completa donde está el ejecutable/script,

    • un indicador de estado (p. ej. Instalado) o un botón para Instalar si falta.

  • Sección “Dependencias”: lista separada con runtimes/dep. (Java, WSL, etc.) y su estado de instalación.

  • Scroll vertical: el panel es desplazable cuando la lista no cabe en la pantalla.

Flujo de uso (paso a paso)

  1. Abrir el panel Configuraciones desde la barra lateral.

  2. Al abrir, el programa ejecuta una comprobación automática de instalación para cada herramienta y dependencia.

  3. Revisar la lista de programas:

    • Si el elemento muestra Instalado, la ruta indicada es donde está el ejecutable.

    • Si el elemento muestra Instalar, pulse ese botón para iniciar el instalador integrado.

  4. Si una dependencia crítica (p. ej. Java o WSL para Kraken2) falta, instálela desde aquí o siga las instrucciones que el instalador muestre.

  5. Tras una instalación correcta el estado se actualizará a Instalado y la ruta aparecerá completa.

Componentes — descripción detallada

  • Manual de Usuario (botón “Abrir”)

    • Función: abre la Support Window en la sección del manual.

    • Recomendación: úsalo para consultar procedimientos o soluciones antes de instalar/desinstalar.

  • Fila de cada programa

    • Nombre e icono: identificación de la herramienta.

    • Descripción breve: para qué sirve (p. ej. “Herramienta de control de calidad”).

    • Ruta en disco: ruta absoluta al ejecutable o script que la aplicación utiliza.

    • Estado / Acción: etiqueta Instalado cuando se detecta; botón Instalar si no está presente.

  • Sección Dependencias

    • Lista de runtimes o componentes (Java, WSL, etc.) con su estado; actúe sobre ellos si falta alguno.

Consejos prácticos

  • Ejecuta el instalador como administrador si aparece un error por permisos.

  • Activa WSL antes de instalar Kraken2 (si usas Windows); en el apartado de instalación incluimos los pasos visuales para habilitarlo.

  • Mantén Java actualizado si alguna herramienta (p. ej. Trimmomatic) lo requiere.

  • Reintenta o revisa logs en caso de error: el instalador suele generar logs con detalles de fallos (útiles para soporte).
