4. Habilitar WSL

Requisito previo: necesitas una cuenta con privilegios de administrador en el equipo.

1. Abrir la herramienta de características de Windows

  1. Haz clic en Inicio (o pulsa la tecla Windows).

  2. Escribe Activar o desactivar las características de Windows y pulsa Enter o selecciona la opción del Panel de control.



Se abrirá la ventana Características de Windows.



2. Marcar las características necesarias

En la ventana Características de Windows:

  1. Busca y marca la casilla Subsistema de Windows para Linux (o Windows Subsystem for Linux).

  2. Para usar WSL 2 y mejorar rendimiento, también marca Plataforma de máquina virtual (Virtual Machine Platform).

  3. Opcionalmente (según versión de Windows y necesidad de virtualización) marca Hyper-V o Plataforma de hipervisor de Windows si aparece y la necesitas.

    • Nota: marcar Hyper-V no siempre es obligatorio para WSL 2, pero en algunos equipos/classes de Windows ayuda a habilitar virtualización completa.


Asegúrate de marcar al menos Subsistema de Windows para Linux y Plataforma de máquina virtual.

3. Aplicar cambios y reiniciar

  1. Haz clic en Aceptar.


  1. Windows aplicará los cambios (instalará los componentes necesarios) y te pedirá reiniciar el equipo.

  2. Reinicia el equipo cuando se solicite.



IMPORTANTE: no podrás usar WSL hasta que completes el reinicio.

4. (Opcional) Verificar que WSL está habilitado

Después del reinicio:

  1. Abre Windows PowerShell o Símbolo del sistema como administrador (clic derecho → Ejecutar como administrador).

  2. Ejecuta el comando de verificación:

wsl -l -v

  • Si WSL está habilitado y hay distribuciones instaladas verás la lista y la versión (1 o 2).

  • Si no hay distribuciones instaladas verás un mensaje indicando que no hay distribuciones instaladas (eso es normal).

5. (Opcional) Convertir/usar WSL2 y/o instalar distribución

  • Si quieres que WSL use la versión 2 por defecto (recomendado cuando la máquina lo soporte), en PowerShell/Admin ejecuta:
wsl --set-default-version 2

Nota: en algunos sistemas puede ser necesaria una actualización del kernel de WSL. Si Windows te lo solicita, descarga e instala el WSL2 kernel update desde Microsoft (la ventana de error indica el enlace).

  • Para instalar una distro (Ubuntu, Debian, etc.) abre la Microsoft Store, busca la distribución deseada e instálala, o bien instala desde PowerShell con wsl --install -d <Distro> si tu Windows soporta ese comando.

6. Última verificación

Tras instalar una distro, ejecuta en PowerShell:

wsl -l -v

Deberías ver algo como:

  NAME      STATE           VERSION
* Ubuntu    Running         2

Si aparece VERSION 2, estás usando WSL 2 correctamente.

Solución de problemas rápidos

  • No ves la opción “Subsistema de Windows para Linux” en la lista: asegúrate de tener Windows 10 (actualizado) o Windows 11; si es una versión muy antigua actualiza Windows.

  • La instalación falla o falta soporte mmap/WSL2: revisa Windows Update y aplica las últimas actualizaciones; reinicia y vuelve a intentar.

  • Prefieres línea de comandos: puedes habilitar las características con DISM si la herramienta visual falla:

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

luego reinicia.
